Heuristic 1:
Name:
Encapsulation (data hiding)
Explanation:
Keeping internal data and logic private prevents external classes from depending on implementation details, making the system easier to modify without breaking other parts.

Heuristic 2:
Name:
Keep related data and behavior together
Explanation:
Placing data and the methods that operate on it in the same class improves readability and makes the code easier to understand and maintain.

Heuristic 3:
Name:
Minimize coupling between classes
Explanation:
Reducing dependencies between classes makes the system more flexible and easier to change, since updates in one class are less likely to impact others.
